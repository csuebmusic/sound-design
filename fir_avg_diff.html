moving-average & differentiator FIR filter

The simplest finite impulse response filter: a weighted sum of recent input samples, no feedback. Tap count N sets the sharpness; the sign pattern (all + or alternating +/−) sets the character (lowpass or highpass).

y[n] = (x[n] ± x[n−1] ± x[n−2] ± ... ± x[n−N+1]) · 1/N
▷ multiplier · ± summer (signs alternate in differentiator mode) · z⁻¹ one-sample delay · ● tap / junction

terminology

An FIR filter is defined entirely by its list of coefficients. There is no feedback, so everything the filter can do is visible right there in the tap values.

h[k] · taps (coefficients)

The numbers that weight each delayed input sample. The whole filter is y[n] = h[0]·x[n] + h[1]·x[n−1] + ... + h[N−1]·x[n−N+1]. For the two filters here, every tap has magnitude 1/N; only the signs change between modes. Those two sign patterns are the two modes you can toggle:

moving average mode

All coefficients are +1/N. Averaging consecutive samples smooths out fast wiggles, so this mode acts as a lowpass filter. The N-tap moving average is the single most common FIR design in all of audio DSP.

differentiator mode

Coefficients alternate sign: +1/N, −1/N, +1/N, −1/N, .... At N=2 this is exactly the first-difference approximation of a derivative, y[n] = (x[n] − x[n−1])/2. For larger N it becomes a highpass comb filter, the mirror image of the moving average: kills DC, peaks at Nyquist. The N slider steps in 2s in this mode, because the DC null is only clean for even tap counts (with odd N the unbalanced + and − coefficients leave 1/N of DC leaking through).

N · number of taps

How many recent input samples are combined. Doubling N roughly doubles the sharpness of the filter's rolloff and roughly doubles the delay before output settles. For an FIR structure, N equals the number of coefficients, and N−1 equals the number of z⁻¹ delay elements in the tapped delay line.

FIR · finite impulse response

With no feedback loop, the impulse response is exactly the coefficient list and nothing more. Hit the filter with a single sample and the output is h[0], h[1], h[2], ..., h[N−1], then silence forever. This "finite" property is what distinguishes FIR from IIR, and it guarantees the filter is always stable.

zeros

Points on the z-plane where the filter's output drops to exactly zero. For both filters here, the zeros sit on the unit circle at equally-spaced angles (one zero every 2π/N), which means the filter completely nulls those frequencies. As N grows, the zeros crowd closer together and the filter gets sharper.

under the hood

z-plane · the zeros

What is the z-plane? A 2D map of complex numbers used to visualize what a filter does. The horizontal axis (Re) is the real part of z; the vertical axis (Im) is the imaginary part. The dashed circle is the unit circle, where |z| = 1, and it represents every frequency from DC (at z = +1) around through Nyquist (at z = −1) and back, with frequency tracked by angle. Filters are characterized by poles (where the transfer function blows up) and zeros (where it drops to zero); their positions on this plane encode the filter's entire behavior.

For this FIR filter: there are no interesting poles (just a trivial one at z = 0), only zeros. Moving average and differentiator both place N−1 zeros on the unit circle at angles spaced 2π/N apart, evenly distributed, but each mode leaves one "slot" empty. Moving average leaves no zero at z = +1 (so DC passes through); differentiator with even N leaves no zero at z = −1 (so Nyquist passes through). Same zero set, different gap, and that single empty slot determines the filter's entire character.

Re = real axis · Im = imaginary axis · dashed circle = unit circle (|z| = 1) · pink ring = zero (N−1 total, on the unit circle) · DC = 0 Hz · Nyq = Nyquist (SR/2) · no poles drawn because the FIR has only the trivial pole at z = 0

impulse response = the coefficients

What is an impulse response? The output of the filter when the input is a single sample of value 1 (a click) followed by silence. Time runs left to right along the horizontal axis (n is the sample number); the vertical axis shows the output value at each sample. The impulse response is a complete fingerprint of the filter: knowing it tells you everything the filter can do to any other signal.

For this FIR filter: the impulse response IS the coefficient list, and nothing more. Send a click in and the output is h[0], h[1], h[2], ..., h[N−1], then dead silence. This is exactly what the "finite" in FIR means: unlike IIR filters whose feedback makes impulse responses ring forever, the FIR output is guaranteed to be zero after N samples.

n = sample number (discrete time axis) · teal stems = h[k] values (output sample = coefficient) · moving average: N identical positive stems · differentiator: N stems alternating sign · after sample N−1: output is exactly zero forever

in more depth

  • The frequency response for an N-tap moving average is the Dirichlet kernel: |H(e^jω)| = (1/N) · |sin(Nω/2) / sin(ω/2)|. This gives the characteristic sinc-like shape with a main lobe near DC and a sequence of side lobes. Nulls occur exactly at ω = 2πk/N.
  • The differentiator response is the exact mirror of the moving average around Nyquist/2: |H_diff(e^jω)| = |H_avg(e^j(π−ω))|. Whatever the moving average does near DC, the differentiator does near Nyquist.
  • Because coefficients are symmetric (for moving average) or sign-antisymmetric (for differentiator), both filters have linear phase: every frequency is delayed by the same number of samples. That delay is τ = (N−1)/2 samples, and it's called the group delay. This is a major reason FIR filters are preferred for applications where waveform shape matters (phase-sensitive audio, image processing).
  • Rolloff past the first zero of the moving average is roughly −6 dB/octave initially, softer than a first-order IIR lowpass with the same cutoff. Moving averages trade sharpness for linear phase and trivial stability.
  • The moving average is not the best FIR lowpass you can design. Its side lobes are only about 13 dB below the main lobe, and the transition band is wide. Windowed-sinc designs (Hamming, Blackman) and optimal designs (Parks-McClellan / remez) shape the coefficients more carefully to trade transition-band width for deeper stopband attenuation.
  • Unlike IIR filters, FIR filters are always stable: no feedback means no unbounded growth. The tradeoff is cost: to match an IIR's sharpness, an FIR typically needs many more taps (and therefore more delay and more multiplications per sample).
  • For efficient implementation of symmetric FIRs, you can exploit symmetry to roughly halve the number of multiplies: h[k] = h[N−1−k] means you can add the paired input samples first, then multiply once.

try this

  1. Set mode = moving average. Sweep N from 2 to 32 and watch the impulse response: same stem height (1/N), more stems, wider "box". Notice how the output lingers longer as N grows; that's the filter's memory.
  2. Start playback with mode = moving average, then sweep N from 2 to 32. At N = 2 the noise still sounds broadband; by N = 16 it's noticeably duller; at N = 32 it's quite muffled, the filter is averaging out all the fast wiggles. Classic low-pass smoothing, by ear.
  3. Set mode = differentiator, N = 2. At N = 2 you are hearing a literal first-difference filter: its gain rises linearly with frequency, so the noise comes out bright, thin, and hiss-forward. Now raise N to 4, 8, 16, 32. As N grows the filter stops behaving like a simple derivative and starts behaving like a comb: the nulls crowd together and the surviving content concentrates into resonant peaks near Nyquist. The spectral character shifts from "airy highpass" to "metallic, comb-like" without ever touching a bass knob.
  4. Toggle between moving average and differentiator at N = 16. Exact same 16-tap structure, exact same magnitudes, just sign flips. The audio swaps LPF for HPF. This is the sign-inversion duality that also showed up in the one-pole's +r versus −r.
  5. Watch the z-plane while sweeping N. For moving average, zeros march evenly around the unit circle with a gap at z=1 (DC escapes). For differentiator, the gap moves to z=−1 (Nyquist escapes). Same zero set, different slot left empty.
  6. With mode = moving average and N = 8, note the group delay reading in the metrics: (8−1)/2 = 3.5 samples. This is what linear phase means: every frequency in your input shows up 3.5 samples later, delayed identically, with no frequency-dependent distortion.
